# Regras para o Codex
Antes de escrever código:
Sempre explicar o plano primeiro
Nunca:
* criar arquivos duplicados
* inventar bibliotecas
* mudar arquitetura
* usar valores hardcoded
Sempre:
* reutilizar código existente
* escrever código simples
* seguir arquitetura
* validar entradas
* tratar erros
Cada tarefa deve ser pequena e incremental.
